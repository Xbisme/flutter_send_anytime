# iOS Flavors (dev / prod)

Both platforms now have dev/prod flavors fully wired.

- **Android** ‚Äî `android/app/build.gradle.kts` `productFlavors { dev, prod }`
  (`app.safesend.dev` / `app.safesend`, per-flavor labels, `minSdk 26`).
- **iOS** ‚Äî generated by [`ios/setup_flavors.rb`](../setup_flavors.rb) (uses the
  `xcodeproj` gem) and verified with `xcodebuild -list`:
  - Build configurations: `Debug-dev` / `Release-dev` / `Profile-dev` and
    `Debug-prod` / `Release-prod` / `Profile-prod` (alongside the originals),
    each pointing at `Flutter/Dev.xcconfig` / `Flutter/Prod.xcconfig`.
  - Shared schemes **`dev`** and **`prod`** (the default `Runner` scheme was
    removed ‚Äî only dev/prod remain).
  - Bundle ids: dev `app.safesend.dev`, prod `app.safesend`; display name via
    `CFBundleDisplayName = $(APP_DISPLAY_NAME)` ("Safe Send Dev" / "Safe Send").

## Run

```bash
flutter run --flavor dev  -t lib/main_dev.dart    # Safe Send Dev ¬∑ app.safesend.dev
flutter run --flavor prod -t lib/main_prod.dart   # Safe Send ¬∑ app.safesend
```

VS Code launch configs for all of these live in `.vscode/launch.json`.

## Pieces (all committed)

- `ios/Flutter/{Debug,Release,Profile}-{dev,prod}.xcconfig` ‚Äî each `#include`s
  the matching `Pods-Runner.<mode>-<flavor>.xcconfig` + `Generated.xcconfig`
  and sets `APP_DISPLAY_NAME` (bundle id is pinned in the build config).
- `ios/Podfile` ‚Äî the `project 'Runner', {‚Ä¶}` hash maps every flavor config to
  `:debug`/`:release` so CocoaPods generates their support files + xcfilelists.
- `ios/setup_flavors.rb` ‚Äî adds the 9 build configs + `dev`/`prod` schemes.

> The CocoaPods "did not set the base configuration ‚Ä¶" warnings during
> `pod install` are expected and benign ‚Äî our flavor xcconfigs already
> `#include` the correct `Pods-Runner.<mode>-<flavor>.xcconfig` (verified:
> `PODS_ROOT` resolves per flavor).

## Regenerate (if Xcode config ever drifts)

```bash
ruby ios/setup_flavors.rb     # idempotent: strips + recreates flavor configs/schemes
cd ios && pod install         # regenerates Pods support files for the flavors
```

Re-run after a `flutter create` regen or an accidental project reset.
