### 2026-06-24 — Spec #002 WebRTC Transport & Transfer Protocol Core ✅ IMPLEMENTED

- **Transport engine shipped** (engine-only, no UI): direct P2P file transfer over an encrypted WebRTC `RTCDataChannel`. Lives in `core/{domain/transfer,services/signaling,services/transport,constants,config}`. Single transfer state machine `idle→connecting→handshaking→transferring→done|failed|cancelled` exposed as a broadcast `Stream<TransferSnapshot>` (sole source of truth). Versioned opcode-framed protocol (`[1-byte opcode][payload]`): manifest → accept/reject → fileStart → 16 KiB chunks → fileComplete(SHA-256) → sessionComplete; cancel either side. Sequential multi-file, **fail-fast**, per-file streamed SHA-256 (crypto `startChunkedConversion`), quarantine `.part` → atomic rename with non-overwriting auto-rename, sender backpressure via `bufferedAmount` thresholds (bounded memory), manifest path-traversal rejection.
- **CI-testable without a server or device**: the engine depends on abstract `SignalingChannel` + `PeerConnector`/`DataTransport` seams; in-process **loopback** impls wire two engines together in one process. Real `WebRtcPeerConnector` (flutter_webrtc) compiles and is wired but is validated only by the **deferred two-device smoke test (T050)**.
- **Quality**: `dart analyze lib test` = **0 issues**; `flutter test` = **63 passed** (27 from #001 + 36 new: protocol codec, loopback signaling, manifest validation, single/multi-file round-trip, fail-fast, collision auto-rename, cancel both sides, reject, malformed/path-traversal, integrity mismatch, disconnect, backpressure, bounded-memory, no-bytes-on-signaling); `dart format` clean. Privacy: logs carry only phase/error-type — never file paths, bytes, IPs, or SDP/ICE.
- **Packages** (verified pub.dev 2026-06-24, Constitution XV): `flutter_webrtc` 1.5.2 (data-channel-only → no camera/mic strings; Android minSdk 26 ≥ 23 ✓; iOS 13 ✓), `crypto` 3.0.7 (streamed SHA-256), `uuid` 4.5.3. **Deferred (device-only)**: iOS `pod install` + Android proguard/Java-8 release config, and the two-physical-device smoke (real NAT + >1 GB throughput) — both part of the first on-device build / #003.
